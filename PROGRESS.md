## 核心成果

### 1. 仿真基础设施

- **Packet 数据类** (`packet.py`)：定义数据包结构（ID、权重、内容、到达时间）
- **PoissonTrafficGenerator** (`traffic.py`)：泊松流量生成器，支持多种权重分布
- **NetworkSimulator** (`network_sim.py`)：离散时间仿真环境
- **实验脚本** (`run_experiment.py`)：支持单场景和多场景对比

### 2. 调度算法实现

#### Max-Weight 调度器 (`maxweight_scheduler.py`)
- **原理**：基于 Lyapunov 漂移理论
- **策略**：`priority = queue_length × packet_weight`
- **特点**：实时、动态、自适应

#### NUM 调度器 (`num_scheduler.py`)
- **原理**：网络效用最大化（凸优化）
- **策略**：按最优速率 (high: 2/3C, low: 1/3C) 分配容量
- **特点**：静态、全局最优

## 实验结果

### 测试场景

4 个场景对比测试：

| 场景 | 链路容量 | 到达速率 | 拥塞程度 | 权重分布 |
|------|---------|---------|--------|---------|
| 轻度拥塞 | 50 | 60 | +20% | 双峰 |
| 中度拥塞 | 50 | 75 | +50% | 双峰 |
| 重度拥塞 | 50 | 100 | +100% | 双峰 |
| 均匀权重 | 50 | 60 | +20% | 均匀 |

### 关键发现

#### 场景1：轻度拥塞（双峰权重）
```
Max-Weight:
  平均队列长度: 195.6
  加权吞吐量: 31.36
  丢包率: 16.15%

NUM:
  平均队列长度: 148.7
  加权吞吐量: 28.75
  丢包率: 28.48%

➜ Max-Weight 优势: +9.07%
```

**解释**：Max-Weight 虽然队列稍长，但优先保留高权重包，加权吞吐量更高；NUM 虽然队列更短，但静态分配导致低权重包浪费了有限容量。

#### 场景2：中度拥塞（双峰权重）
```
Max-Weight:
  加权吞吐量: 35.16
  丢包率: 33.23%

NUM:
  加权吞吐量: 34.10
  丢包率: 34.25%

➜ Max-Weight 优势: +3.12%
```

**解释**：拥塞加重时，两者差异缩小。NUM 由于刚好满足容量约束，性能接近。

#### 场景3：重度拥塞（双峰权重）
```
Max-Weight:
  加权吞吐量: 38.81
  丢包率: 49.83%

NUM:
  加权吞吐量: 33.80
  丢包率: 50.70%

➜ Max-Weight 优势: +14.83%  ← 最大优势！
```

**重要发现**：在极限拥塞下，Max-Weight 的动态适应能力显著优于 NUM 的静态策略。

#### 场景4：均匀权重（轻度拥塞）
```
两个算法性能接近（都无权重分化）
```

**结论**：权重分布的多样性对 Max-Weight 算法的收益影响很大。

## 关键洞察

### 1. 权重分布的影响

- **双峰分布**（30% 重要 + 70% 不重要）：Max-Weight **显著优势** (+9% ~ +15%)
- **均匀分布**：两者差异不大

**原因**：Max-Weight 的核心优势在于实时感知队列压力，优先保留高权重包。权重差异越大，这个优势越明显。

### 2. 拥塞程度的影响

- **轻度 → 中度**：优势从 9% 缩小到 3%
- **中度 → 重度**：优势突然增大到 15%

**原因**：
- 轻→中：容量约束变紧，NUM 的优化开始生效
- 中→重：NUM 无法再有效控制队列，Max-Weight 的实时调整能力显得更重要

### 3. 队列稳定性

- **Max-Weight**：队列稳定在 ~195-200
- **NUM**：队列稳定在 ~148-150

**启示**：Max-Weight 允许更大的队列换取更高的加权吞吐量，这对语义通信场景是合理的（只要我们关心的是"有用信息通过量"而非"包数"）。

## 如何运行

### 单次实验
```bash
python3 run_experiment.py
# 按 Enter 选择默认（单个基准实验）
```

### 多场景对比
```bash
echo "2" | python3 run_experiment.py
# 自动运行 4 个场景，输出对比结果
```

### 修改参数
编辑 `run_experiment.py` 中的 `scenarios` 列表：
```python
scenarios = [
    {
        "name": "自定义场景",
        "link_capacity": 50,      # 修改容量
        "arrival_rate": 80,        # 修改到达率
        "weight_dist": "bimodal"   # 修改权重分布
    }
]
```
